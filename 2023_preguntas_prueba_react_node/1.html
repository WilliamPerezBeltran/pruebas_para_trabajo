You are given an array arr of num distinct integers. Your task is to determine whether the array can be sorted in increasing order using exactly one move, as described below:

  • One move consists of choosing a valid index k (0<=k<num-1) to cut the array and then pasting the first subarray arr[0,k] after the second subarray arr[k+1, num].


Return the results as follows:

  • If it is possible to sort the array arr in increasing order using one move as described above, return the index k
  • If it is not possible to sort the array arr in increasing order using one move, return -1
  • If the array arr is already sorted in increasing order without performing a move, return 100


Take the following into account:

  • 1 <= num <= 65
  • 0 <= arr[i] <= 99 ( 0 <= i < num)
  • All the elements of the array arr are distinct

 


Example 1

Input 1 (num): 6

Input 2 (arr): 8, 10, 15, 35, 3, 7

Output: 3

 

Explanation: 

If we choose k=3, the subarray[0,3] of the array will be: 8,10,15,35. After pasting this subarray after the remaining subarray (3, 7), the array will be sorterd in increasing order: 3,7,8,10,15,35. We should therefore return k, which is 3.



Example 2

Input 1 (num): 6

Input 2 (arr): 9, 10, 3, 4, 7, 50

Output: -1


Explanation:

No matter where we cut the array arr, it is not possible to sort it in an increasing order using one more. The answer, therefore, is -1.
